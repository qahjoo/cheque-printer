# 🧾 نظام الشيكات — شهد وهبة للتمور / ميلانو للتمور

تطبيق سطح مكتب (Electron) لإدارة وطباعة الشيكات، مع تذكيرات متعددة القنوات،
تقارير، ومزامنة سحابية عبر Supabase. الواجهة عربية بالكامل (RTL) بخط Cairo.

---

## المميزات الرئيسية

- ➕ **إدارة الشيكات**: إضافة/تعديل، تفقيط تلقائي للمبلغ إلى كلمات عربية، تغيير
  الحالة (مفتوح / محصّل / مرتجع / ملغي)، حذف ناعم مع الاحتفاظ بالسجل.
- 🖨️ **طباعة الشيكات**: محرر قوالب بالسحب والإفلات لكل بنك (بالمليمتر)، ومعاينة
  بالحجم الفعلي قبل الطباعة عبر طابعة النظام.
- 📊 **لوحة تحكم**: مؤشرات لكل حالة، تنبيهات الشيكات المتأخرة، والمستحقة قريباً.
- 🔔 **تذكيرات**: عبر إشعارات سطح المكتب، تيليغرام، والبريد الإلكتروني — تعمل
  تلقائياً عدة مرات يومياً ضمن ساعات محددة، مع رسالة واحدة مجمّعة لكل القنوات.
- 📈 **تقارير**: مستحقات الأسبوع، كشف حساب بالمستفيد، تقرير فترة زمنية، وملخص
  الوضع الحالي — مع تصدير PDF و Excel وطباعة.
- ☁️ **مزامنة سحابية**: كل تغيير يُرفع إلى Supabase؛ عند انقطاع الاتصال يُوضع في
  طابور ويُعاد المحاولة تلقائياً (آخر كتابة تفوز).
- 📜 **سجل مراجعة** غير قابل للتعديل أو الحذف (محمي بمُشغّلات قاعدة البيانات).
- 🔒 **قفل بالرقم السري** (bcrypt) مع قفل مؤقت بعد 5 محاولات خاطئة.
- 💾 **نسخ احتياطي** تصدير/استيراد JSON مع دمج بدون تكرار.

## المتطلبات

- **Node.js 18** أو أحدث (يوفّر `fetch` المستخدَم في تيليغرام).
- **npm**.
- أدوات بناء لحزمة `better-sqlite3` الأصلية:
  - Windows: `npm install --global windows-build-tools` أو تثبيت "Desktop
    development with C++" من Visual Studio Build Tools.
- حساب **Supabase** (مجاني) للمزامنة.

## التثبيت والتشغيل (خطوة بخطوة)

```bash
# 1) تثبيت الحزم
npm install

# 2) تجهيز ملف البيئة
cp .env.example .env
#   ثم افتح .env واملأ SUPABASE_URL و SUPABASE_ANON_KEY على الأقل.
#   (يمكن تركها فارغة وسيفتح التطبيق "معالج الإعداد" لإدخالها.)

# 3) تجهيز قاعدة Supabase (مرة واحدة)
#   افتح Supabase SQL Editor والصق محتوى الملف supabase_setup.sql ثم شغّله.

# 4) التشغيل في وضع التطوير (Vite + Electron معاً)
npm run dev

# أو التشغيل من نسخة مبنية:
npm run build
npm start
```

### بناء نسخة قابلة للتوزيع (Windows)

```bash
npm run dist
# تُنتَج نسخة التثبيت في مجلد release/
```

> **الخطوط والأيقونات**: لطباعة PDF عربي صحيح، ضع ملف الخط في
> `build/fonts/Cairo-Regular.ttf`. لأيقونة النظام ضع `build/tray.png` و
> `build/icon.ico`. التطبيق يعمل بدونها لكن نص PDF العربي يتحول إلى `?`.

## الاستخدام اليومي

1. **مراجعة الصباح**: افتح **لوحة التحكم** لرؤية المتأخرات والمستحقات القريبة،
   أو **التقارير → مستحقة هذا الأسبوع**.
2. **إضافة شيك**: من الشريط الجانبي **➕ شيك جديد** — املأ البيانات، ويظهر
   التفقيط تلقائياً، ثم احفظ.
3. **الطباعة**: من **سجل الشيكات** اضغط 🖨️ بجانب الشيك، أو من شاشة التعديل
   **معاينة الطباعة**.
4. **تحصيل شيك**: افتح الشيك ← اختر الحالة **محصّل** ← أدخل اسم المحصِّل.
5. **التذكيرات**: تعمل تلقائياً. لتشغيلها يدوياً استخدم **🔔 التذكيرات → تشغيل
   الآن** أو من أيقونة النظام (Tray) **تذكيرات الآن**.

## الإعدادات

من **⚙️ الإعدادات**:

- **عام**: اسم الشركة، الشعار، أسماء العملة، تنسيق التاريخ.
- **التذكيرات**: أيام التنبيه المسبق، عدد التذكيرات يومياً، ساعات العمل، وتفعيل
  القنوات. زر **اختبار التذكيرات الآن** و**سجل التذكيرات**.
- **البنوك**: إضافة/تعديل/حذف البنوك ومقاسات الشيكات، وتحرير قالب الطباعة.
- **المزامنة**: حالة Supabase وعدد عناصر الطابور، ومزامنة يدوية.
- **النسخ الاحتياطي**: تصدير/استيراد JSON.
- **الأمان**: تغيير الرقم السري، قفل التطبيق.

> **مهم**: تُحفظ الإعدادات العامة في قاعدة البيانات، أمّا القيم السرية (التوكنات،
> كلمات المرور، مفاتيح Supabase) فتبقى في ملف `.env` **فقط**.

## البنية التقنية

```
database/migrations/   ← مخطط SQLite + بيانات تجريبية
main/                  ← عملية Electron الرئيسية
  main.js              ← الإقلاع، النوافذ، Tray
  db.js                ← الوصول لقاعدة البيانات + المزامنة + سجل المراجعة
  ipc/                 ← نطاقات IPC (شيكات، بنوك، إعدادات، تقارير...)
  services/            ← Supabase, التذكيرات, تيليغرام, البريد, الطباعة
preload.js             ← جسر contextBridge الآمن (window.api)
renderer/              ← واجهة React (Vite + Tailwind RTL)
  screens/ utils/ components/ lib/
supabase_setup.sql     ← مخطط Supabase + سياسات RLS
```

- **الطباعة والتصدير** تُنفَّذ في العملية الرئيسية؛ الواجهة تُشغّلها فقط عبر IPC.
- **مصدر الحقيقة محلياً** هو SQLite؛ Supabase للنسخ السحابي والوصول عن بُعد.

## استكشاف الأخطاء وإصلاحها

| المشكلة | الحل |
|---|---|
| فشل `npm install` عند `better-sqlite3` | ثبّت أدوات بناء C++ (Visual Studio Build Tools)، ثم `npm run postinstall`. |
| "غير مهيأ" في تبويب المزامنة | تأكد من `SUPABASE_URL` و`SUPABASE_ANON_KEY` في `.env` وأعد التشغيل. |
| التذكيرات لا تصل عبر تيليغرام | تحقق من `TELEGRAM_BOT_TOKEN` و`TELEGRAM_CHAT_ID`، وأرسل رسالة للبوت أولاً. |
| نص PDF العربي يظهر `?` | ضع `build/fonts/Cairo-Regular.ttf` — يُضمَّن الخط تلقائياً. |
| نسيت الرقم السري | لا يوجد استرجاع؛ يمكن مسح `pin_hash` من جدول `settings` في ملف قاعدة البيانات داخل مجلد بيانات المستخدم. |
| التطبيق "يُغلق" لكنه يبقى | هذا مقصود — يُصغَّر إلى شريط النظام. للإغلاق الكامل: أيقونة النظام ← **إغلاق التطبيق**. |

### مواقع الملفات

- قاعدة البيانات: داخل مجلد بيانات المستخدم في Electron
  (`%APPDATA%/ChequePrinter` على Windows).
- ملف `.env`: بجانب التطبيق (أو جذر المشروع أثناء التطوير).

## الأمان والخصوصية

- سجل المراجعة **غير قابل للتعديل أو الحذف** (مُشغّلات SQLite تمنع UPDATE/DELETE).
- الرقم السري يُخزَّن كـ bcrypt hash.
- الأسرار لا تُخزَّن في قاعدة البيانات ولا تُسجَّل في سجل المراجعة (تُقنَّع).
- تُفتح الروابط الخارجية في متصفح النظام لا داخل التطبيق.
